# iTax PoC

A proof-of-concept tool for managing Korean NTS (국세청) corporate tax forms — upload, fill, calculate, version, and print them.

## What it does

| Feature | Description |
|---|---|
| **Form upload** | Admin uploads an NTS tax form PDF (and optionally the matching Excel file from Hancom). Claude AI extracts all field definitions, labels, section groupings, and formula relationships. |
| **Field extraction** | Claude reads each PDF page and returns structured field metadata: field ID, label, type (`input` / `calculated`), section, and formula string. |
| **Formula engine** | Calculated fields are evaluated client-side using a two-pass arithmetic evaluator that mirrors the NTS form logic (e.g. field `10 = 01 + 02 - 03`). |
| **Form versioning** | Each upload creates a new version of a form. The previous version is automatically marked `is_current = false`. A migration map (added/removed/changed fields) is generated by Claude. |
| **Company management** | Companies are registered with name and business registration number, then linked to form entries. |
| **Entry editing** | Users create a form entry for a company and fiscal year, enter input values, and see calculated totals update in real time. |
| **Print preview** | Table-format print view shows all fields by section with calculated values highlighted. Uses the browser print dialog. |
| **Bulk upload** | Multiple PDFs can be uploaded in one request (up to 10 parallel Claude calls). |

## Local setup

### Prerequisites

- Python 3.11+
- Node.js 18+
- A [Supabase](https://supabase.com) project
- An [Anthropic API key](https://console.anthropic.com)

### 1. Backend

```bash
cd backend
cp .env.example .env
# Edit .env with your Supabase URL, service role key, and Anthropic API key
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### 2. Frontend

```bash
cd frontend
npm install
npm run dev
```

The frontend dev server proxies `/api` to `http://localhost:8000`.

### 3. Database

Run the SQL scripts in `supabase/` against your Supabase project in order:

1. `migrations.sql` — core tables
2. `add_companies.sql`
3. `add_pdf_hash.sql`
4. `migrate_form_entries.sql`

## Limitations (PoC scope)

- Single-tenant (no auth / user accounts)
- Print view is table format only — does not reproduce the original NTS form layout
- Formula syntax must match NTS field numbering exactly
- Excel bbox extraction (for overlay view) is not reliable across all NTS form layouts
